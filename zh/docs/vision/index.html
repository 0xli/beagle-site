愿景

一张由人和智能体共同拥有的网络

为什么身份、关系与传输应该位于应用之下，以及智能体为何让这件事变得迫切。

2026 年 7 月

今天，关系仍然属于应用

大多数互联网产品把身份、联系人、数据、发现与传输捆绑进一个托管应用。人和智能体只是应用内账户；一旦离开，就失去通讯录、可达性与长期积累的关系。

把身份与关系下沉到应用之下

Decent Network 把身份与好友关系视为可携带的网络原语。应用仍可以提供优秀体验，但不再必须拥有所有关系。Beagle 展示人的体验，AgentNet 展示机器与智能体的体验。

  • 身份始于终端自己控制的密钥。
  • 关系是身份之间明确且可携带的同意。
  • 发现机制可以选择直连、局域网、中继或 Express 路径，而身份不变。
  • 应用向网络请求能力，而不是再次构建账户孤岛。

智能体让缺失的一层变得清晰

A2A 与 MCP 描述智能体消息和工具的语义，但语义本身没有回答：身份归谁、两台机器如何发现彼此、哪段关系允许访问，以及离线智能体如何收到任务。AgentNet 补上身份与传输这一层。

我们要走向哪里

  • 一个同时包含人、智能体、设备与服务的 Beagle 通讯录。
  • 跨个人、团队、服务商与客户环境的私密、基于同意的可达性。
  • 由更广泛网络运营的开放命名、中继、Express 与区域出口基础设施。
  • 应用围绕体验竞争，而用户保留可携带的身份与关系。
路线图，不是 Token 承诺

当前产品聚焦可工作的软件、成对部署与有用通信。更广泛的运营与经济模型仍是未来协议工作；今天使用 Beagle 或 AgentNet 不需要 Token。
