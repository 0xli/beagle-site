安装

在 Windows 上原生安装(免 WSL)

直接在 64 位 Windows 上运行 Beagle —— 守护进程、代理路由器、网页控制台,不需要 WSL。

2026 年 7 月

三个组件

该用哪个 Windows 指南?

这份原生指南是大多数人的推荐路径 —— 直接在 Windows 上运行 Beagle,不需要额外学习任何东西。另一份 WSL2 指南面向已经在 Windows 上运行 Linux 环境的开发者,或企业策略禁止原生模式的情况。

Beagle 可直接运行在 64 位 Windows 上。npm 包已经包含原生 helper 和 WireGuard 官方签名的 wintun.dll,不需要另外下载网卡驱动。

组件作用运行方式地址
守护进程 (agentnet up)加入 Decent Network 并创建 agentnet0 虚拟网卡Windows 启动任务 (SYSTEM)后台
路由器 (agentnet proxy router)本地多出口代理和国内分流;可选 HLS 加速PowerShell(当前用户)127.0.0.1:8889
界面 (agentnet ui)好友、消息和文件的网页控制台PowerShell(当前用户)http://localhost:8765
系统要求

Windows 10/11 或 Windows Server 2016+,64 位 x64 或 ARM64,以及 Node.js 20+。不支持 Windows Server 2008 R2/2012 R2 和 Node.js 12。只有在企业策略阻止原生模式时才使用 Windows (WSL) 指南。

1. 安装 Node.js

Beagle 通过 npm 分发 —— npm 是随 Node.js 一起安装的包管理器,而 Node.js 是 OpenJS 基金会的免费开源运行环境。你只需安装一次 Node.js,就得到了下面用到的 npm 和 agentnet 命令。不需要会编程也能完成这些步骤。

01
下载 LTS 安装包

打开 nodejs.org,下载当前的 Windows LTS 安装包,用默认选项完成安装 —— 保留勾选 Add to PATH。

02
在新的 PowerShell 中检查

关闭并重新打开 PowerShell,检查版本。node --version 必须显示 v20.x 或更高。

node --version
npm --version

2. 安装 Beagle 并创建身份

打开管理员 PowerShell(开始 → 输入 PowerShell → 右键 → 以管理员身份运行),然后执行:

npm install -g @decentnetwork/lan
agentnet --version
agentnet init --name my-windows-pc
密钥属于你自己

agentnet init 会创建加密身份 —— 不需要账号、邮箱或密码。所有文件位于 $env:USERPROFILE\.agentnet,密钥在 carrier\keypair.json。请备份整个目录,绝不要把密钥发给别人。

3. 安装并启动守护进程

继续在管理员 PowerShell 中执行。这条命令会安装签名的 Wintun 虚拟网卡、10.86.0.0/16 路由,并创建名为 Decent AgentNet 的高权限启动任务,随开机启动、异常退出后自动重启。

agentnet service install

检查运行状态,第一次启动后请等待 60–90 秒让 dora 名单和节点会话完成收敛:

agentnet service status
agentnet diag
Get-NetAdapter -Name agentnet0
Get-NetRoute -DestinationPrefix 10.86.0.0/16

4. 启动代理路由器

另开一个普通 PowerShell 窗口。普通浏览只需启动路由器并保持窗口不关闭。它监听 127.0.0.1:8889,按 Ctrl+C 停止。普通代理、聊天和网页界面都不需要安装 CA 证书。

agentnet proxy router

可选 —— 观看 CCTV 等 HLS 视频:先用 Ctrl+C 停止普通路由器,只需信任一次本机视频 CA(Windows 会弹出安全确认,请允许),再带加速参数启动。该 CA 只在本机生成,只用于指定的国内视频 CDN 域名。

agentnet proxy trust-ca
agentnet proxy router --hls-accel

5. 启动网页界面

再开一个普通 PowerShell 窗口并保持运行,然后浏览器打开 http://localhost:8765 添加好友、聊天和发送文件。按 Ctrl+C 停止界面。

agentnet ui

6. 让 Chrome 使用代理

运行下面的命令,打开一个单独使用 Beagle 的 Chrome,平时的 Chrome 不受影响。默认使用国内分流:匹配的国内网站和视频走可用出口,其他流量直接连接。

& "$env:ProgramFiles\Google\Chrome\Application\chrome.exe" `
  --proxy-server="127.0.0.1:8889" `
  --user-data-dir="$env:TEMP\chrome-beagle" `
  --proxy-bypass-list="<-loopback>"
或使用插件

也可以安装 ZeroOmega 或 FoxyProxy,添加 HTTP 代理 127.0.0.1 端口 8889。想让所有网站都走出口?用 agentnet proxy router --all 启动;只有需要 HLS 视频加速时才加 --hls-accel。

管理、升级、移除

以下守护进程命令需要在管理员 PowerShell 中运行。路由器和界面就是两个普通 PowerShell 窗口,用 Ctrl+C 停止,重新执行相应命令即可再次启动。

agentnet service status
agentnet service restart
Get-Content -Wait "$env:USERPROFILE\.agentnet\agentnet-service.log"

升级 Beagle,然后分别重启路由器和界面窗口,确保三个组件都用新版本:

npm install -g @decentnetwork/lan@latest
agentnet service restart
彻底移除守护进程

只有彻底卸载时才运行:agentnet service uninstall。

可选:把这台电脑作为出口节点

在管理员 PowerShell 中运行。出口代理默认监听本机 AgentNet IP 的 8888 端口。--hls-accel 应运行在观看视频的客户端上,不需要加到每个出口节点。

agentnet proxy enable
agentnet service restart

疑难排查

现象解决
Unexpected token '?' 或 engine 不支持Node.js 太旧。安装 Node.js 20+。
找不到 agentnet 命令安装 Node.js/软件包后重开 PowerShell。确认 %APPDATA%\npm 已加入 PATH。
Access is denied / 路由或 Wintun 网卡失败以管理员身份重新打开 PowerShell,再执行 agentnet service install。
提示 Missing TUN helper 或缺少 wintun.dll重新安装 @decentnetwork/lan,并检查杀毒软件是否隔离了 bin\windows-* 下的文件。不要从第三方网站下载 DLL。
agentnet diag 连接不到守护进程执行 agentnet service restart,再检查 agentnet-service.log。
刚启动时看不到节点等待 60–90 秒。如果仍未恢复,确认其他 VPN 没有创建冲突的 10.86.* 路由。
浏览器无法通过 127.0.0.1:8889 打开网页确认守护进程和 agentnet proxy router 窗口都在运行。
视频能播放但仍停顿确认路由器带有 --hls-accel,且 agentnet proxy trust-ca 已成功。
企业策略禁止原生模式使用 WSL 指南。不要让原生和 WSL 守护进程同时使用同一个身份。
