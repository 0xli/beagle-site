连接机器与私有服务

通过身份门控的私有地址继续使用现有应用，而不开放公网端口。

2026 年 7 月

访问模型

AgentNet 分配 10.86.x.x 私有地址，优先寻找直连路径；网络无法直连时使用中继。好友关系是第一层信任边界，具体服务权限仍是另一项需要明确决定的配置。

  • 不需要公网 IP 或路由器端口转发。
  • 陌生身份不会自动获得路由能力。
  • 现有 SSH、HTTP、RDP、TCP 与 UDP 软件继续使用原协议。
  • 撤销关系即可移除身份级路径，无需重新设计两边网络。

基本流程

01
在两台机器安装

按照平台指南安装，并用 `agentnet doctor` 检查。

02
交换身份

发送并接受好友请求。

03
查看路由

在 Desktop 或 CLI 中找到对方的私有地址。

04
访问目标服务

服务与权限配置完成后，用原有客户端连接。

智能区域代理

本地代理可以让特定区域目的地经过已批准出口，其余流量保持直连。默认安装使用中国区分流；全局隧道是需要主动选择的模式。
