Decent Network 白皮书

协议主张、架构、信任模型与网络路线图的产品化可读版本。

2026 年 7 月

摘要

Decent Network 提出一套自组织通信层：终端拥有身份、直接建立关系、发现可达路径并交换加密流量，而不要求某个应用提供商拥有整张网络。Beagle 与 AgentNet 是建立在这一主张上的参考产品。

核心原语

原语作用
身份由本地控制的密钥对，以及人、智能体或设备的稳定地址。
好友关系身份之间的双向同意，建立关系与第一层信任边界。
发现DHT 与辅助服务定位当前路径，但不改变地址。
传输加密直连、局域网路径、中继回退、离线投递与可选 Express 路线。
命名可解析到可携带身份的人类可读名称。

参考架构

  • 终端生成并保留身份材料。
  • Bootstrap 服务帮助新终端进入发现网络。
  • DHT 记录帮助 peer 找到当前可达信息。
  • NAT 穿透优先建立直连，中继提供回退。
  • Express 与区域出口为性能或策略需求提供可选路径。
  • Beagle 等应用消费身份、关系、消息与网络能力。

信任与安全模型

自主身份不再把平台账户当作信任根，但并不会消除运维安全。用户仍需保护终端密钥、核对重要关系、更新软件、限制本地控制台，并且只开放真正需要的服务。好友关系赋予一段关系，不等于无限管理员权限。

基础设施与未来经济模型

协议设计预期 Bootstrap、中继、Express、命名与出口服务由独立参与者运营。早期白皮书曾探讨激励机制与 Token 门控服务角色；当前产品不需要 Token。运营经济与治理仍是需要在真实网络需求之后验证的路线图课题。

今天已经有什么

  • Beagle 移动通信与身份体验。
  • Beagle Desktop 与 AgentNet CLI/TUI 的好友、消息、文件、路由、出口与诊断。
  • 10.86.x.x 私有网络、支持中继的可达性与智能区域代理。
  • 面向智能体与应用集成的 OpenClaw Channel 与 TypeScript 软件包。
