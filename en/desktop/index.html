// complete feature set

What the desktop and AgentNet stack actually do.

This is not only a chat window. It is the user-facing control plane for a private identity network.

Friend and identity management
Generate your address locally, copy or share it, send requests, accept or reject, rename, remove, and see presence.
Persistent messaging
Conversation history survives restarts, unread state is tracked, and delivery path shows whether a message arrived live or offline.
Native file exchange
Drag files into chat; monitor speed and progress; cancel, resume, download, delete, or preview media inline.
Private virtual LAN
Trusted peers receive private 10.86.x.x addresses so existing TCP/UDP services work without public IPs or port forwarding.
Smart regional proxy
Use a China split tunnel by default, full tunnel when needed, or choose specific regional exits.
Route and exit visibility
Inspect peer paths, virtual IPs, wire versions, direct/relay state, active exits, and dora allocations.
Interactive terminal UI
Run agentnet console for an always-open friend list and conversation pane with keyboard controls.
Automation-ready CLI
Script friend requests, messages, file sends, diagnostics, routing, grants, revocation, and service operations.
// terminal

Use the same network without leaving your shell.

The interactive TUI is for humans who live in a terminal. The one-shot CLI is for agents, scripts, support workflows, and automation.

agentnet consoleopen the interactive friend list and chat
agentnet send <peer> <text>send an encrypted message
agentnet file <peer> <path>send a file over the native exchange path
agentnet whoshow the identity you can share
agentnet doctordiagnose the local stack

Beagle is the product. AgentNet is the engine.

01Beagle Desktop

The local web console for people: chat, files, friends, profiles, routes, and exits.

The machine-side runtime: identity, persistence, private LAN, access policy, proxy routing, and automation.

03Decent Network

The underlying identity, friendship, discovery, encrypted transport, relay, express, and naming infrastructure.
